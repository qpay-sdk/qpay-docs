# QPay SDK

## QPay V2 Payment API SDK for every language

Integrate QPay payments into your application with official SDKs for 11 languages. Each SDK provides automatic token management, typed request/response models, and comprehensive error handling.

---

## Supported Languages

| Language | Package | Install |
|---|---|---|
| **Go** | `qpay-go` | `go get github.com/usukhbayar-boop/qpay-go` |
| **JavaScript / TypeScript** | `qpay-js` | `npm install qpay-js` |
| **Python** | `qpay-py` | `pip install qpay-py` |
| **PHP** | `qpay-php` | `composer require usukhbayar/qpay-php` |
| **Java** | `qpay-java` | Maven / Gradle (see docs) |
| **Ruby** | `qpay` | `gem install qpay` |
| **.NET (C#)** | `QPay` | `dotnet add package QPay` |
| **Dart / Flutter** | `qpay` | `dart pub add qpay` |
| **Swift** | `QPay` | Swift Package Manager |
| **Rust** | `qpay` | `cargo add qpay` |
| **cURL** | `qpay-curl` | Shell scripts (no install) |

## Key Features

All SDKs share the same design principles:

- **Automatic Token Management** -- Tokens are obtained, cached, and refreshed automatically. You never need to manage authentication manually.
- **Type-safe Models** -- Strongly typed request and response objects for all endpoints.
- **Comprehensive Error Handling** -- Structured error types with QPay error codes, HTTP status codes, and raw response bodies.
- **Full API Coverage** -- Invoices, payments, ebarimt (electronic tax receipts), and all QPay V2 endpoints.
- **Environment Variable Config** -- Load configuration from environment variables or configure manually.
